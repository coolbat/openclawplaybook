## When to use this template

Use this when the team already has logs, but the logs are too noisy to support fast triage. The template turns repeated failures into manageable clusters.

## Minimal setup

Start with one log stream and one target error pattern. Do not mix several unrelated systems in the first version.

## What good output looks like

The output should cluster similar failures, preserve examples, and suggest where to look next. It should help a responder focus before opening raw logs.

## First extension

Add routing by severity or subsystem once the clustering quality feels accurate enough to trust.
