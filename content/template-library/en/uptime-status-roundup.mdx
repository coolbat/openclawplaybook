## When to use this template

Use this when people need a regular service status view but do not want to open dashboards for a quick answer. It is a translation layer from metrics to status.

## Minimal setup

Pick one monitoring source and one reporting interval. The first pass should summarize health rather than explain every metric in detail.

## What good output looks like

The summary should show what stayed healthy, what degraded, and whether any action is needed. It should be readable by people outside the immediate ops team.

## First extension

Add a simple trend comparison with the previous period to make the roundup more decision-friendly.
