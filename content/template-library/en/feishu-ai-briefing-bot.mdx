## When to use this template

Use this when your team already lives in Feishu and new workflows only stick when they feel native. The template is designed to fit an existing communication habit instead of asking the team to adopt a new surface.

## Minimal setup

Start with one Feishu destination and one recurring input source. Do not over-design the first version. The key is proving that structured updates arrive where the team already pays attention.

## What good output looks like

A good Feishu briefing is short, legible, and easy to act on from within the chat flow. If people need to open three extra tools to understand it, the template is not doing enough.

## First extension

Add role-specific sections only after the shared team update format feels stable. That prevents the bot from becoming a noisy internal newsletter.
