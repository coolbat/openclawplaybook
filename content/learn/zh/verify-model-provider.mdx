## 先把变量缩到最少

首次运行失败时，新手常常会同时去查渠道、提示词、自动化逻辑和模型配置。这样只会让问题更乱。更好的做法，是先单独证明一个模型服务本身是健康的。

## 最小验证目标

你至少要确认三件事：

- API key 是有效的
- base URL 是对的
- 模型能返回一条正常响应

只要这三点成立，后续很多不确定性就被排除了。

## 为什么这一步要放前面

很多看起来像工作流问题、渠道问题的现象，根源其实是模型访问本身失败。越早确认 provider，越少走弯路。

> **只先接一个 provider**
>   在第一个 provider 跑通之前，不要同时接多个服务。额外选项只会增加噪音。
